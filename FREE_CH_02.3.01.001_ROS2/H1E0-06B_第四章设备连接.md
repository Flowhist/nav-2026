# H1E0-06B 单线激光雷达：第四章 设备连接整理

> 来源：H1E0-06B 产品说明书 V1.1 第 4 章“设备连接”。

---

## 4.1 设备连接

### 所需配件

完成设备连接并输出扫描区域内目标物体距离信息，至少需要：

- 12–26V 直流电源，典型值为 24V；
- H1E0-06B 单线激光雷达；
- 装有 TCP/IP 协议的终端。

> 注意：直流电源供电电压禁止超过 30V。

### 连接步骤

1. 将 H1E0-06B 单线激光雷达放置在合适位置，使其能够扫描目标区域。
2. 连接主要接口：
   - 网口：使用适配网线，通过 TCP Client 连接协议终端 TCP Server；
   - 电源接口：使用适配电源线连接 12–26V 直流电源。
3. 直流电源输出供电，雷达自检后开始工作。
4. 通过指令与雷达通信。

### 默认网络连接信息

| 项目 | 默认值 |
|---|---:|
| IP | `192.168.1.111` |
| 子网掩码 | `255.255.255.0` |
| 网关 | `192.168.1.1` |
| 端口 | `2111` |

## 4.2 与设备通信

### 指令基本格式

H1E0-06B 指令由以下部分组成：

```text
文本开始 + 文本长度 + 操作码 + 指令号 + 参数 + 文本结束
```

| 字段 | 说明 | 长度/字节 | Hex |
|---|---|---:|---|
| 文本开始 | 起始报头 | 4 | `02 02 02 02` |
| 文本长度 | 从文本开始到文本结束的总长度 | 2 | `XX XX` |
| 操作码 | 读 / 写 / 方法 | 1 | 见下表 |
| 指令号 | 每个指令号代表一个指令 | 1 | `XX` |
| 参数 | 0–9 个参数，长度不定 | X | `XX ... XX` |
| 文本结束 | 校验和：文本结束前所有字节求和，取低 8 bit | 1 | `XX` |

### 操作码定义

| 操作类型 | 发送方 | 接收方回复 |
|---|---:|---:|
| 读 | `0x00` | `0x10` |
| 写 | `0x01` | `0x11` |
| 方法 | `0x02` | `0x12` |

---

## 4.2.1 登录

应用程序以客户端身份登录接入雷达通信。若不执行登录，或登录密码错误，雷达将拒绝执行修改配置相关指令。出厂默认登录密码为：

```text
F4 72 47 44
```

> 登录密码可根据指令修改，但测试阶段暂不开放此功能。

### 发送登录指令

| 字段 | Hex |
|---|---|
| 文本开始 | `02 02 02 02` |
| 文本长度 | `00 0E` |
| 操作码 | `02` |
| 指令号 | `01` |
| 用户权限 | `03` |
| 密码 | `F4 72 47 44` |
| 校验和 | `26` |

说明书示例发送帧：

```text
02 02 02 02 00 0E 02 01 03 F4 72 47 44 0D
```

### 回复

```text
登录成功：02 02 02 02 00 0A 12 01 01 26
登录失败：02 02 02 02 00 0A 12 01 00 25
```

```text
发送：02 02 02 02 00 09 02 02 15

登出成功：02 02 02 02 00 0A 12 02 01 27
登出失败：02 02 02 02 00 0A 12 02 00 26
```

## 4.2.3 设置雷达 IP 地址

设置雷达 IP 地址。后续支持 IP 地址和网关前三段联动更改。

| 字段 | Hex / 说明 |
|---|---|
| 文本开始 | `02 02 02 02` |
| 文本长度 | `00 0D` |
| 操作码 | `01` |
| 指令号 | `0F` |
| 参数 | 要设置的 IP 地址，4 字节 |
| 校验和 | `XX` |

示例：将 IP 设置为 `192.168.0.2`。

```text
发送：02 02 02 02 00 0D 01 0F C0 A8 00 02 8F
回复：02 02 02 02 00 0D 11 0F C0 A8 00 02 9F
```

若设置失败，会返回雷达原自身 IP 地址。

## 4.2.14 设置转速角分辨率

| 参数 | 长度/字节 | 说明 | 示例 |
|---|---:|---|---|

## 4.2.22 请求单次数据

注意事项：

- 雷达转速被配置后，约十几秒内无法响应应用程序的数据请求；
- 雷达重启或断电重新上电后，30 秒内无法响应数据请求；
- 单次数据请求返回上一次扫描获得的数据，因此输出数据与请求时刻可能存在延时；
- 以 15Hz 为例，最大延时可能为 `66ms@15Hz`；
- 单次数据会分包发送。

### 单次数据请求

```text
发送：02 02 02 02 00 09 02 30 43
```

### 单次数据回复格式

| 参数 | 长度/字节 | 说明 |
|---|---:|---|
| 参数 1 | 1 | 设备状态：忙碌 `00`，就绪 `01`，错误 `02` |
| 参数 2 | 2 | 扫描次数，范围 `0–65535` |
| 参数 3 | 1 | 报文编号 |
| 参数 4 | 2 | 扫描频率 × 100，例如 15Hz 为 `05 DC`，25Hz 为 `09 C4` |
| 参数 5 | 2 | 角分辨率 × 10000，例如 0.33° 为 `0D 05` |
| 参数 6 | 2 | 一帧数据点云数，例如 0.33°、-45°–225°范围下为 811 |
| 参数 7 | 2 | 当前包首个点索引 |
| 参数 8 | 2 | 包内点数 N |
| 参数 9 | 4×N | 扫描数据：每个点包含距离 2 字节、反射率 2 字节 |
| 参数 10 | 10 | 时间戳：年 2 字节，月/日/时/分/秒各 1 字节，微秒 3 字节 |

回复总长度：

```text
33 + 4 × N
```

## 4.2.23 连续请求数据开关

参数取值：

| 功能 | Hex |
|---|---:|
| 开启连续取数 | `01` |
| 关闭连续取数 | `00` |

### 开关指令示例

```text
开启连续取数发送：02 02 02 02 00 0A 02 31 01 46
关闭连续取数发送：02 02 02 02 00 0A 02 31 00 45

连续取数已开启回复：02 02 02 02 00 0A 12 31 01 56
连续取数已关闭回复：02 02 02 02 00 0A 12 31 00 55
```

### 连续数据格式

雷达回复连续请求数据开关指令后，会发送连续数据，连续数据也会分包发送。连续数据的指令号为：

```text
0x32
```

连续数据参数格式与单次数据基本一致：

| 参数 | 长度/字节 | 说明 |
|---|---:|---|
| 参数 1 | 1 | 设备状态：忙碌 `00`，就绪 `01`，错误 `02` |
| 参数 2 | 2 | 扫描次数，范围 `0–65535` |
| 参数 3 | 1 | 报文编号 |
| 参数 4 | 2 | 扫描频率 × 100 |
| 参数 5 | 2 | 角分辨率 × 10000 |
| 参数 6 | 2 | 一帧数据点云数 |
| 参数 7 | 2 | 当前包首个点索引 |
| 参数 8 | 2 | 包内点数 N |
| 参数 9 | 4×N | 距离 2 字节 + 反射率 2 字节 |
| 参数 10 | 10 | 时间戳 |

回复总长度：

```text
33 + 4 × N
```

## 附：常用通信流程建议

```text
1. TCP 连接雷达：192.168.1.111:2111
2. 发送登录指令
3. 查询设备状态，确认设备就绪
4. 根据需要读取或设置 IP / 网关 / 子网掩码 / 扫描角度 / 转速角分辨率等参数
5. 如修改配置，发送保存用户参数指令
6. 请求单次数据，或开启连续取数
7. 使用完毕后，可发送登出指令
```
